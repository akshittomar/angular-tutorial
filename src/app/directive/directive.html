Directives are classes in Angular that add behavior to elements in the DOM. They allow developers to encapsulate DOM manipulation, conditional rendering, and dynamic styling in a declarative way using templates. Directives are a core feature of Angular for extending HTML with custom behavior and for composing reusable UI logic.

Primary Types of Directives

1. Structural Directives: These directives change the structure of the DOM by adding, removing, or manipulating elements. They are used to control which elements are rendered based on application state or data. e.g: ngIf , ngFor , ngSwitch.

2. Attribute Directives: Attribute directives modify the appearance or behavior of an existing element, component, or another directive. They do not add or remove DOM elements but change properties, styles, classes, or behavior of the host element. e.g: ngClass , ngStyle.

3. Component Directives: Components are a specialized kind of directive with their own template. A component combines an HTML template with a class that manages its data and behavior; components are the primary building blocks of Angular applications. e.g: Any Angular Component that starts with @Component decorator.
