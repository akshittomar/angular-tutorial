Signal Tutorial

What is a Signal? A signal is a special kind of reactive value that notifies the system when it changes, allowing automatic updates to parts of the application that depend on it.

Signal vs Normal Value: Signals trigger effect() functions to run whenever their value changes, enabling reactive updates. In contrast, normal values do not trigger effect() and changes to them do not automatically update dependent parts of the application.

There are 2 types of Signals : 1. Computed Signal 2 . Wrtitable Signal

1.Computed Signal : this is a signal that cannot be changed directly (they don't have set or update property). It is derived from other signals or values.

Each effect only reacts to the signals used inside its own function. If you have an effect that only uses signal a, it will only run when a changes. If you have another effect that only uses signal b, it will only run when b changes. Signal a cannot trigger the effect for signal b, and vice versa. Each effect is independent and only tracks the signals it accesses.
