Injectable kya hota hai?
@Injectable() Angular ko batata hai ki ye class ek service hai aur isko components ya other services mein inject kiya ja sakta hai.

providedIn ka matlab:
Jab hum providedIn: 'root' likhte hain, to service poori app ke liye ek hi baar banti hai aur sab components usi instance ko use karte hain.

providers kya hota hai?
providers array mein service dene ka matlab hota hai ki sirf us component ke liye service ka naya instance banega. Baaki components us service ko share nahi karenge.

providedIn vs providers:
providedIn: 'root' → service poori app mein share hoti hai.
providers → service sirf ek component/module ke liye hoti hai.

Real projects mein kya use hota hai?
Mostly projects mein providedIn: 'root' use hota hai kyunki ye clean, simple aur best practice hai.

Service ko use kaise karein? (Step-by-step)

Step 1: Service file banao aur usme logic likho (jaise Product service).

Step 2: Component file mein service ko constructor ke through inject karo.

Code Example (Component - TypeScript):
constructor(productService: Product) {
  this.product = productService.getProducts();
}

Step 3: Ab component ke variable ko template mein use karo.

Code Example (Template - HTML):
@for(prod of product; track prod){
  { { prod.name }} ,
  { { prod.discount }} ,
  { { prod.cost }}
}

One-line summary:
Service mein logic/data hota hai, component usko constructor ke through inject karta hai, aur template us data ko display karta hai.
