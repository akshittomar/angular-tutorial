Styling in Angular

  • Component-Level CSS (Encapsulation)
    • Each Angular component can have its own CSS file (e.g., parent.component.css, child.component.css).
    • The styles defined in a parent component’s CSS do not affect the child component’s styles.
    • Each component’s styleUrls property controls which CSS files apply to that component only.
  • Global CSS
    • To apply styles globally (across all components), edit the src/styles.css file.
    • To add multiple global CSS files, update the styles array in angular.json:
      • "styles": [
  "src/styles.css",
  "src/global2.css"
]
  • Multiple CSS Files in a Component
    • You can import multiple CSS files in a component by adding their paths to the styleUrls array:
      • @Component({{ '{' }}
  styleUrls: ['./comp1.css', './comp2.css']
{{ '}' }})
  • Internal CSS (Within a Component)
    • You can write internal CSS using the <style> tag directly in the component’s template:
      • <style>
  .my-class {{ '{' }} color: red; {{ '}' }}
</style>
  • Inline CSS (HTML Attribute)
    • You can use the style attribute for inline CSS on any HTML element:
      • <div style="color: blue;">Inline styled text</div>
  • Inline Styles in TypeScript
    • You can define small CSS rules directly in the component’s decorator using the styles property:
      • @Component({{ '{' }}
  styles: ['.special {{ '{' }} font-weight: bold; {{ '}' }}']
{{ '}' }})

Summary:
Angular supports component-scoped CSS, global CSS, multiple CSS files per component, internal and inline styles, and even inline styles in TypeScript. Parent and child component styles are encapsulated by default, so they don’t interfere with each other. For global styles, use styles.css or update the styles array in angular.json.
